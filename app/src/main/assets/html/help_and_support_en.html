Playback
External Players If the external player has been started but no video is showing, check the following points:
  • The selected playback profile is not suitable. Recommended is 'matroska'.
  • The streaming port (default is 9981) is not correct.
  • The player can reach the server via the streaming port. Maybe the port is not properly forwarded by the router or a proxy.
  • Uploading the data from the server takes too long
Internal Player If the internal player has been started but no video is showing, check the following:
  • The selected playback profile is not suitable. Recommended is 'htsp'.
  • Uploading the data from the server takes too long
Casting
Requirements
  • You have purchased the full version of TVHClient.
  • You need transcoding support working on the TVH server side.
  • Transcoding, especially VAAPI hardware-assisted transcoding, works best on recent unstable TVH 4.3 versions.
Setup Transcoding

For maximum compatibility (Casting, most browsers, players and the internal player) use a stream profile with the following settings:

  • Container: av-lib/MP4 (example)
  • Video codec: H.264, High profile (example)
  • Audio codec: AAC LC profile (example)
Setup TVHClient
  • You can either use an IP-address or a DNS name for the connection. Before casting is started the DNS name will be converted to an IP-address so that casting will work.
  • Select the appropriate casting playback profile.
